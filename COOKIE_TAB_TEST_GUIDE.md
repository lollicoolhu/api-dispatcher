# Cookie Tab 分组功能 - 测试指南

## 快速测试

### 前提条件
✅ 服务器已启动：http://localhost:3000
✅ 管理面板：http://localhost:3000/admin

### 测试步骤

#### 步骤1：创建测试数据

1. 访问管理面板：http://localhost:3000/admin
2. 在"访问日志"中找到一个接口（如 `/api/v2/account/info`）
3. 点击"修改返回"

#### 步骤2：创建第一个用户的配置

1. 在弹窗中点击"+ 添加条件"
2. 设置条件：
   - 来源：Cookie
   - 键：`userId`
   - 操作符：等于
   - 值：`123`
3. 编辑响应内容（如：`{"user": "User 123"}`）
4. 在"备注"中输入：`用户123`
5. 点击"临时保存"

#### 步骤3：创建第二个用户的配置

1. 再次点击"修改返回"（或在临时修改列表中点击"管理"）
2. 点击"新建版本"
3. 点击"+ 添加条件"
4. 设置条件：
   - 来源：Cookie
   - 键：`userId`
   - 操作符：等于
   - 值：`456`
5. 编辑响应内容（如：`{"user": "User 456"}`）
6. 在"备注"中输入：`用户456`
7. 点击"临时保存"

#### 步骤4：创建第三个用户的配置

1. 重复步骤3，但使用：
   - 值：`789`
   - 响应：`{"user": "User 789"}`
   - 备注：`用户789`

#### 步骤5：创建无Cookie条件的配置

1. 再次点击"修改返回"
2. 点击"新建版本"
3. **不添加任何条件**
4. 编辑响应内容（如：`{"user": "Default User"}`）
5. 在"备注"中输入：`默认用户`
6. 点击"临时保存"

#### 步骤6：查看 Tab 分组

1. 返回"设置"标签页
2. 滚动到"临时修改列表"区域
3. 你应该看到以下 Tab：
   ```
   [全部 (1)]  [无Cookie条件 (1)]  [userId=123 (1)]  [userId=456 (1)]  [userId=789 (1)]
   ```

#### 步骤7：测试 Tab 切换

1. 点击 `userId=123` Tab
   - 应该只显示包含 `userId=123` 条件的配置
   - 其他配置被隐藏

2. 点击 `userId=456` Tab
   - 应该只显示包含 `userId=456` 条件的配置

3. 点击 `无Cookie条件` Tab
   - 应该只显示没有 Cookie 条件的配置（默认用户）

4. 点击 `全部` Tab
   - 应该显示所有配置

#### 步骤8：测试实际请求

使用 curl 测试不同用户的响应：

```bash
# 测试用户123
curl -H "Cookie: userId=123" http://localhost:3000/api/v2/account/info

# 测试用户456
curl -H "Cookie: userId=456" http://localhost:3000/api/v2/account/info

# 测试用户789
curl -H "Cookie: userId=789" http://localhost:3000/api/v2/account/info

# 测试默认用户（无Cookie）
curl http://localhost:3000/api/v2/account/info
```

## 故障排查

### 问题1：Tab 没有显示

**可能原因**：
- 只有少于3个分组（"全部"和"无Cookie条件"）
- 没有任何 Cookie 条件的配置

**解决方案**：
- 创建至少一个有 Cookie 条件的配置

### 问题2：Tab 切换无效

**可能原因**：
- JavaScript 错误
- 浏览器缓存问题

**解决方案**：
- 打开浏览器开发者工具，查看 Console 是否有错误
- 清除浏览器缓存，刷新页面

### 问题3：配置没有出现在预期的 Tab 中

**可能原因**：
- Cookie 条件设置不正确
- 条件的 `source` 不是 `cookie`

**解决方案**：
- 检查配置的条件设置
- 确保条件来源选择的是 "Cookie"
