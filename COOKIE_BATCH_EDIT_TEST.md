# Cookie 批量修改功能 - 测试指南

## 测试准备

### 1. 启动服务器

```bash
node lib/server-core.js
```

### 2. 访问管理页面

打开浏览器：`http://localhost:3000/admin`

## 测试用例

### 测试用例 1：批量修改优先级

#### 准备数据

创建 3 个临时修改，都使用相同的 Cookie 条件：

**临时修改 A**
- 路径：`/api/user/info`
- Cookie 条件：`userId=17700001818`
- 优先级：1
- 备注：用户信息

**临时修改 B**
- 路径：`/api/user/balance`
- Cookie 条件：`userId=17700001818`
- 优先级：1
- 备注：账户余额

**临时修改 C**
- 路径：`/api/user/orders`
- Cookie 条件：`userId=17700001818`
- 优先级：1
- 备注：订单列表

#### 测试步骤

1. 切换到 `userId=...1818` Tab
2. 验证显示 3 个临时修改
3. 点击"批量修改"按钮
4. 验证弹窗显示：`将修改 3 个临时修改`
5. 验证列表显示 3 个临时修改的路径和备注
6. 选择"修改优先级"
7. 输入 `10`
8. 点击"批量修改"
9. 确认对话框点击"确定"

#### 预期结果

- 显示成功提示：`成功批量修改 3 个临时修改`
- 列表自动刷新
- 所有 3 个临时修改的优先级都变为 10
- 在每个临时修改的信息区域显示：`优先级: 10`

### 测试用例 5：多个 Cookie 分组

#### 准备数据

除了测试用例 1 的数据，再创建 2 个使用不同 Cookie 条件的临时修改：

**临时修改 D**
- 路径：`/api/admin/users`
- Cookie 条件：`token=admin_abc123`
- 优先级：1
- 备注：管理员用户列表

**临时修改 E**
- 路径：`/api/admin/settings`
- Cookie 条件：`token=admin_abc123`
- 优先级：1
- 备注：管理员设置

#### 测试步骤

1. 验证显示 2 个 Cookie Tab：
   - `userId=...1818 (3)`
   - `token=...c123 (2)`
2. 切换到 `token=...c123` Tab
3. 验证只显示 2 个临时修改（D 和 E）
4. 点击"批量修改"按钮
5. 验证弹窗显示：`将修改 2 个临时修改`
6. 选择"修改优先级"
7. 输入 `20`
8. 点击"批量修改"并确认

#### 预期结果

- 只有临时修改 D 和 E 的优先级变为 20
- 临时修改 A、B、C 的优先级保持不变（仍为 10）
- 切换回 `userId=...1818` Tab，验证 A、B、C 未受影响

## 边界情况测试

### 边界 1：空分组

**场景**：删除某个 Cookie 分组的所有临时修改

**预期**：
- Tab 自动消失（因为没有临时修改了）
- 自动切换到"全部" Tab

### 边界 2：单个临时修改

**场景**：Cookie 分组只有 1 个临时修改

**预期**：
- 仍然显示"批量修改"按钮
- 弹窗显示：`将修改 1 个临时修改`
- 可以正常批量修改

### 边界 3：大量临时修改

**场景**：Cookie 分组有 50 个临时修改

**预期**：
- 弹窗列表可以滚动查看
- 批量修改可能需要几秒钟
- 所有临时修改都被正确更新
