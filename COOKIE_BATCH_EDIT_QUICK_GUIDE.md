# Cookie 批量修改 - 快速使用指南

## 功能说明

批量修改当前 Cookie 分组下所有临时修改的 Cookie 条件值。

## 使用步骤

### 1. 切换到 Cookie Tab

在临时修改列表中，点击要修改的 Cookie 分组 Tab。

例如：`userId=...1818`

### 2. 点击"批量修改"

Tab 导航栏右侧的黄色按钮。

### 3. 输入新值

在"新的 Cookie Value"输入框中输入新的值。

例如：将 `17700001818` 改为 `17700001819`

### 4. 确认修改

点击"批量修改"按钮，在确认对话框中点击"确定"。

### 5. 完成

所有临时修改的 Cookie 条件值已更新，列表自动刷新。

## 典型场景

### 场景 1：切换测试账号

**需求**：从测试账号 A 切换到测试账号 B

**操作**：
1. 切换到账号 A 的 Tab（如 `userId=...1818`）
2. 点击"批量修改"
3. 输入账号 B 的 ID（如 `17700001819`）
4. 确认修改

**结果**：所有接口的 Mock 数据自动切换到账号 B

### 场景 2：更新 Token

**需求**：Token 过期，需要更新所有使用该 Token 的接口

**操作**：
1. 切换到 Token 的 Tab（如 `token=...abc123`）
2. 点击"批量修改"
3. 输入新的 Token 值
4. 确认修改

**结果**：所有接口的 Token 条件已更新

### 场景 3：环境切换

**需求**：从开发环境切换到测试环境

**操作**：
1. 切换到 `env=dev` Tab
2. 点击"批量修改"
3. 输入 `test`
4. 确认修改

**结果**：所有接口自动切换到测试环境

## 注意事项

### 1. 不可撤销

批量修改操作会立即生效并保存，无法撤销。

**建议**：修改前仔细检查受影响的版本列表。

### 2. Tab 会变化

修改后，原来的 Tab 会消失（因为没有临时修改使用旧值了），新的 Tab 会出现。

**示例**：
- 修改前：`userId=...1818` Tab（10个临时修改）
- 修改后：`userId=...1819` Tab（10个临时修改）

### 3. 只修改 Cookie 值

只修改 Cookie 的值（value），不修改键名（key）。

**示例**：
- 修改前：`userId=17700001818`
- 修改后：`userId=17700001819`
- Cookie Key 保持不变：`userId`

### 4. 同时修改所有匹配的条件

如果一个临时修改有多个 Cookie 条件，只有匹配当前分组的条件会被修改。

**示例**：
- 临时修改有两个条件：`userId=17700001818` 和 `env=dev`
- 在 `userId=...1818` Tab 中批量修改
- 只有 `userId` 条件会被修改，`env` 条件保持不变

## 常见问题

### Q: 如何批量修改 Cookie Key？

A: 目前不支持批量修改 Cookie Key，只能修改 Value。如需修改 Key，请逐个编辑。

### Q: 可以批量修改其他类型的条件吗（如 Header、Body）？

A: 目前只支持批量修改 Cookie 条件。其他类型的条件需要逐个编辑。

### Q: 修改后原来的 Tab 消失了，如何找回？

A: 修改后会自动创建新的 Tab（使用新的 Cookie 值）。如果需要恢复，可以再次批量修改回原来的值。

### Q: 批量修改会影响其他 Cookie 分组吗？

A: 不会。批量修改只影响当前选中的 Cookie 分组，其他分组不受影响。

### Q: 如何批量删除某个 Cookie 分组的所有临时修改？

A: 目前不支持批量删除，需要逐个删除，或直接编辑配置文件。
