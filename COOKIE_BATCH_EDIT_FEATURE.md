# Cookie 分组批量修改功能

## 功能概述

在 Cookie Tab 分组视图中，可以一键批量修改当前分组下所有临时修改的 **Cookie 条件值**。

## 使用场景

当你有多个临时修改都使用相同的 Cookie 条件时（例如 `userId=17700001818`），可以：
- 批量将 Cookie 值改为另一个值（例如改为 `userId=17700001819`）
- 快速切换测试账号
- 统一更新 Token 值

## 典型应用

### 场景 1：切换测试账号

你为测试账号 A（`userId=17700001818`）创建了 20 个接口的 Mock 数据，现在想切换到测试账号 B（`userId=17700001819`）：

**传统方式**：逐个编辑 20 个临时修改，手动修改 Cookie 值

**批量修改**：
1. 切换到 `userId=...1818` Tab
2. 点击"批量修改"
3. 输入新值 `17700001819`
4. 一键完成所有修改

### 场景 2：更新 Token

Token 过期后需要更新所有使用该 Token 的临时修改：

**传统方式**：逐个查找并修改

**批量修改**：
1. 切换到对应的 Token Tab
2. 点击"批量修改"
3. 输入新的 Token 值
4. 批量更新完成

### 场景 3：环境切换

从开发环境（`env=dev`）切换到测试环境（`env=test`）：

**批量修改**：
1. 切换到 `env=dev` Tab
2. 点击"批量修改"
3. 输入 `test`
4. 所有接口自动切换到测试环境

## 功能入口

### 1. 切换到 Cookie 分组 Tab

在"设置"页面的"临时修改列表"中，当存在 Cookie 条件的临时修改时，会自动显示 Tab 分组。

### 2. 点击"批量修改"按钮

在任意 Cookie 分组 Tab（除了"全部"和"无Cookie条件"）中，Tab 导航栏右侧会显示"批量修改"按钮。

**注意**：
- "全部" Tab 不显示批量修改按钮（因为包含所有类型的临时修改）
- "无Cookie条件" Tab 不显示批量修改按钮（这些临时修改没有 Cookie 条件）

## 批量修改弹窗

### 弹窗内容

1. **受影响的版本数量**
   - 显示当前分组中有多少个临时修改会被修改
   - 蓝色提示框突出显示

2. **受影响版本列表**
   - 显示所有将被修改的临时修改
   - 格式：`路径 - 备注名称`
   - 可滚动查看（最多显示200px高度）

3. **当前 Cookie 条件**
   - 黄色提示框显示当前的 Cookie 条件
   - 格式：`key=value`
   - 只读显示，不可修改

4. **Cookie Key（只读）**
   - 显示 Cookie 的键名
   - 灰色背景，不可编辑

5. **当前 Cookie Value（只读）**
   - 显示当前的 Cookie 值
   - 灰色背景，不可编辑

6. **新的 Cookie Value（必填）**
   - 输入新的 Cookie 值
   - 黄色边框突出显示
   - 所有临时修改的 Cookie 条件值将被更新为此值

### 操作流程

#### 完整流程示例

假设你有一个测试账号 `userId=17700001818`，为它创建了 10 个不同接口的临时修改。

**步骤 1**：切换到对应的 Cookie Tab
- 点击 `userId=...1818` Tab（或你设置的自定义标签）

**步骤 2**：点击"批量修改"按钮
- Tab 导航栏右侧的黄色按钮

**步骤 3**：查看受影响的版本
- 弹窗显示：`将修改 10 个临时修改的 Cookie 条件`
- 列表显示所有 10 个临时修改的路径和名称

**步骤 4**：查看当前 Cookie 条件
- 黄色提示框显示：`userId=17700001818`
- Cookie Key 显示：`userId`（只读）
- 当前 Cookie Value 显示：`17700001818`（只读）

**步骤 5**：输入新的 Cookie 值
- 在"新的 Cookie Value"输入框中输入：`17700001819`

**步骤 6**：确认修改
- 点击"批量修改"按钮
- 弹出确认对话框：
  ```
  确定要将 10 个临时修改的 Cookie 条件
  从 userId=17700001818
  改为 userId=17700001819 吗？
  ```
- 点击"确定"

**步骤 7**：完成
- 系统自动更新所有临时修改的 Cookie 条件
- 显示成功提示：`成功批量修改 10 个临时修改的 Cookie 条件`
- 列表自动刷新
- 原来的 `userId=...1818` Tab 消失（因为没有临时修改使用这个条件了）
- 出现新的 `userId=...1819` Tab（包含所有 10 个临时修改）

#### 3. 启用/禁用

**用途**：批量启用或禁用所有临时修改

**选项**：
- 启用：所有临时修改将被启用
- 禁用：所有临时修改将被禁用

**示例**：
- 临时禁用测试账号的所有 Mock 数据
- 批量启用生产账号的所有临时修改

## 技术实现

### 前端逻辑

#### 1. 识别受影响的版本

```javascript
function batchEditCookieGroup() {
  const paths = Object.keys(overrides);
  const affectedVersions = [];
  
  paths.forEach(p => {
    const versions = overrides[p] || [];
    versions.forEach(v => {
      const cookieConditions = (v.conditions || []).filter(c => c.source === 'cookie');
      cookieConditions.forEach(cc => {
        const groupKey = `cookie:${cc.key}=${cc.value}`;
        if (groupKey === activeOverrideCookieTab) {
          affectedVersions.push({ path: p, version: v });
        }
      });
    });
  });
  
  openBatchEditModal(affectedVersions);
}
```

#### 2. 批量更新

```javascript
async function saveBatchEdit() {
  for (const { path, version } of affectedVersions) {
    await fetch('/admin/overrides', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        path,
        versionId: version.id,
        ...updateData
      })
    });
  }
  
  await loadOverrides();
  renderOverrides();
}
```

### 后端 API

使用现有的 `/admin/overrides` POST 接口：

**请求参数**：
- `path`: 接口路径
- `versionId`: 版本 ID
- `priority`: 优先级（可选）
- `delay`: 延迟时间（可选）
- `enabled`: 启用状态（可选）

**特点**：
- 支持部分更新（只更新提供的字段）
- 自动保存到配置文件
- 返回更新结果

## 注意事项

### 1. 操作不可撤销

批量修改操作会立即生效并保存到配置文件，无法撤销。建议：
- 修改前仔细检查受影响的版本列表
- 重要操作前备份配置文件

### 2. 优先级冲突

批量修改优先级后，可能导致：
- 某些临时修改被其他更高优先级的规则阻挡
- 列表中显示"不可达"标记

建议：
- 了解优先级规则（本地文件夹=0，全局服务器=100）
- 合理设置优先级避免冲突

### 3. 性能考虑

批量修改会逐个发送请求到服务器：
- 修改数量较多时可能需要几秒钟
- 修改过程中请勿关闭页面
- 完成后会自动刷新列表

### 4. Cookie 条件匹配

只有包含完全相同 Cookie 条件的临时修改才会被批量修改：
- `userId=123` 和 `userId=456` 是不同的分组
- 同一个版本可能有多个 Cookie 条件，只要有一个匹配就会被包含

## 使用建议

### 1. 按用户分组管理

为不同的测试账号创建不同的 Cookie 条件：
- 测试账号A：`userId=17700001818`
- 测试账号B：`userId=17700001819`
- 管理员账号：`token=admin_token_xxx`

然后可以：
- 批量调整某个账号的所有接口优先级
- 批量禁用某个账号的所有 Mock 数据
- 批量设置某个账号的响应延迟

### 2. 环境切换

使用 Cookie 条件区分不同环境：
- 开发环境：`env=dev`
- 测试环境：`env=test`
- 生产环境：`env=prod`

然后可以：
- 批量启用/禁用某个环境的所有 Mock
- 批量调整某个环境的优先级

### 3. 功能测试

为不同的功能测试场景创建分组：
- 正常流程：`scenario=normal`
- 异常流程：`scenario=error`
- 边界情况：`scenario=edge`

然后可以：
- 快速切换测试场景（批量启用/禁用）
- 统一调整某个场景的响应延迟

## 故障排除

### Q: 点击"批量修改"没有反应？

A: 检查：
1. 是否在 Cookie 分组 Tab 中（不是"全部"或"无Cookie条件"）
2. 当前分组是否有临时修改
3. 浏览器控制台是否有 JavaScript 错误

### Q: 批量修改后部分版本没有更新？

A: 可能原因：
1. 网络请求失败（检查浏览器控制台）
2. 服务器返回错误（检查服务器日志）
3. 版本 ID 不存在（可能已被删除）

### Q: 批量修改后列表显示不正确？

A: 尝试：
1. 手动刷新页面（F5）
2. 检查配置文件是否正确保存
3. 重启服务器

### Q: 如何批量删除某个分组的所有临时修改？

A: 目前不支持批量删除，需要：
1. 先批量禁用（避免影响其他功能）
2. 然后逐个删除，或直接编辑配置文件
