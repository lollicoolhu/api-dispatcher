## 功能说明

### 1. 临时修改列表
- **复选框**: 启用/禁用该接口的临时修改（启用最后一个启用的版本）
- **接口路径**: 显示完整的API路径
- **版本数量**: 显示该接口有多少个版本（如果>1）
- **备注标签**: 显示当前启用版本的备注
- **优先级**: 显示当前启用版本的优先级
- **编辑按钮**: 打开版本管理弹窗
- **删除全部**: 删除该接口的所有版本

### 2. 版本管理弹窗
- **新建版本按钮**: 为该接口创建新的临时修改版本
- **版本列表**: 显示所有版本，默认折叠
- **版本头部**: 点击可展开/折叠版本详情
  - 箭头图标（▶/▼）
  - 版本序号
  - 启用状态（●已启用 / ○未启用）
  - 创建时间
  - 单选按钮（选择要启用的版本）
  - 编辑按钮
  - 删除按钮
- **版本详情**（展开后显示）:
  - 优先级
  - 备注
  - 内容预览（最多10行，超过显示省略）

### 3. 版本折叠/展开
- 点击版本头部可展开/折叠版本详情
- 箭头图标会旋转90度表示展开状态
- 默认所有版本都是折叠状态

### 4. 版本单选
- 每个版本前有单选按钮
- 选中某个版本会自动启用该版本，禁用其他版本
- 当前启用的版本单选按钮默认选中

## 使用流程

### 创建新版本
1. 点击列表中的"编辑"按钮打开版本管理弹窗
2. 点击"+ 新建版本"按钮
3. 在编辑窗口中输入内容、优先级、备注
4. 点击"临时保存"
5. 新版本自动启用，弹窗关闭

### 查看版本详情
1. 点击列表中的"编辑"按钮打开版本管理弹窗
2. 点击要查看的版本头部
3. 版本详情展开，显示优先级、备注和内容预览
4. 再次点击版本头部可折叠

### 切换版本
1. 打开版本管理弹窗
2. 点击要启用的版本前的单选按钮
3. 该版本自动启用，其他版本自动禁用
4. 弹窗自动关闭

### 编辑版本
1. 打开版本管理弹窗
2. 点击要编辑的版本的"编辑"按钮
3. 修改内容、优先级或备注
4. 点击"临时保存"
5. 版本更新，保持原有启用状态

### 删除版本
1. 打开版本管理弹窗
2. 点击要删除的版本的"删除"按钮
3. 确认删除操作
4. 如果还有其他版本，弹窗保持打开并刷新
5. 如果是最后一个版本，弹窗关闭

### 删除全部版本
1. 点击列表中的"删除全部"按钮
2. 确认删除操作
3. 该接口的所有临时修改版本被清除

## 优先级规则

临时修改的优先级遵循以下规则：

1. **本地文件夹** (优先级 ≤ 0)
2. **临时修改** (优先级 1-99)
3. **URL映射** (优先级 1-99)
4. **文件夹映射** (优先级 1-99)
5. **全局服务器** (优先级 ≥ 100)

同优先级时按类型排序：
- override > exactMapping > localFolder > wildcardMapping > folderMapping > globalServer

## 注意事项

1. **唯一启用**: 每个接口同时只能有一个启用的版本
2. **自动禁用**: 启用新版本时会自动禁用其他版本
3. **版本ID**: 每个版本有唯一的ID（时间戳+随机字符串）
4. **创建时间**: 版本按创建时间排序，最新的在前
5. **JSON验证**: 编辑时会验证JSON格式（仅对.json文件）
6. **不可达检测**: 自动检测是否被更高优先级规则阻挡
7. **折叠状态**: 已启用的版本默认展开，未启用的版本默认折叠
8. **内容预览**: 版本详情中只显示前10行内容，超过显示省略
9. **自适应高度**: 列表和详情面板高度根据窗口大小自动调整
10. **日志中的版本**: 日志详情中修改接口时也会显示版本管理弹窗

## 技术细节

### 数据结构
```json
{
  "tempOverrides": {
    "/api/path": [
      {
        "id": "1773307540545_9wwp65j4w",
        "content": "{\"test\":true}",
        "enabled": true,
        "priority": 1,
        "remark": "测试版本",
        "createdAt": "2026-03-12T09:25:40.545Z"
      }
    ]
  }
}
```

### API接口
- `GET /admin/overrides`: 获取所有临时修改
- `POST /admin/overrides`: 创建或更新版本
  - 不指定 `versionId`: 创建新版本
  - 指定 `versionId`: 更新现有版本
- `DELETE /admin/overrides`: 删除版本
  - 指定 `versionId`: 删除单个版本
  - 不指定 `versionId`: 删除全部版本

### 前端函数
- `renderOverrides()`: 渲染临时修改列表
- `editOverride(path)`: 打开版本管理弹窗
- `openOverrideVersionModal(path, versions)`: 显示版本管理弹窗
- `toggleVersionAccordion(versionKey)`: 切换版本折叠/展开
- `selectOverrideVersion(path, versionId)`: 选择并启用版本
- `editOverrideVersion(path, versionId)`: 编辑指定版本
- `deleteOverrideVersionInModal(path, versionId)`: 删除指定版本
- `createNewOverrideVersion(path)`: 创建新版本
- `closeOverrideVersionModal()`: 关闭版本管理弹窗
